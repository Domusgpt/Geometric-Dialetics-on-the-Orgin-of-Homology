# Paper Outline Generator

Generate structured outlines for academic papers on Polytopal Projection Processing.

## Paper Types

### 1. Full Research Paper (8-12 pages)
- Abstract (150-250 words)
- Introduction (1-1.5 pages)
- Related Work (1-1.5 pages)
- Method/Framework (2-3 pages)
- Experiments/Evaluation (2-3 pages)
- Discussion (0.5-1 page)
- Conclusion (0.5 page)
- References

### 2. Short Paper / Workshop (4-6 pages)
- Abstract (100-150 words)
- Introduction + Related Work (1.5 pages)
- Method (1.5-2 pages)
- Preliminary Results (1 page)
- Conclusion + Future Work (0.5 page)

### 3. Position Paper / Vision (2-4 pages)
- Abstract
- The Problem
- Our Vision
- Key Insights
- Research Agenda
- Conclusion

### 4. Technical Report / White Paper (10-20 pages)
- Executive Summary
- Problem Statement
- Technical Approach
- Architecture
- Implementation
- Evaluation
- Deployment Considerations
- Roadmap

## Task

Generate an outline for the paper type and topic specified. Include:

1. **Section headers** with target word counts
2. **Key points** to cover in each section
3. **Figure/table suggestions** with captions
4. **Potential reviewer concerns** to preempt
5. **Citation placeholders** for key references

## Output Format

```
# [Paper Title]

## Target Venue: [venue name]
## Paper Type: [type]
## Target Length: [pages/words]

---

## Abstract (~200 words)
- Hook: [opening angle]
- Problem: [what gap we address]
- Approach: [our solution]
- Results: [key findings]
- Impact: [why it matters]

## 1. Introduction (~800 words)
### 1.1 Opening Hook
- [specific point]

### 1.2 Problem Statement
- [specific point]

[Continue for all sections...]

---

## Figures & Tables
1. **Figure 1**: [description] - illustrates [concept]
2. **Table 1**: [description] - compares [things]

## Preemptive Responses
- Concern: [anticipated criticism]
  Response: [how we address it]

## Key Citations Needed
- [topic]: [citation placeholder]
```
